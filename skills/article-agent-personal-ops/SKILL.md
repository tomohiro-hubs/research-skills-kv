# 個人エージェントは「作り方」より「運用の型」が先

このスキルは、パーソナルAI/エージェントの作成技術よりも、その「権限設定」や「検査設計」といった運用ルールに焦点を当てた記事執筆を支援します。

## 記事の構成要素 (Core Components)

### 1. タイトル / アングル (Title/Angle)
- **個人エージェントは「作り方」より「運用の型」が先**
- 狙い: AIの民主化に伴い、誰もが「自分だけのAI」を作れるようになるからこそ、「どう飼いならすか」が重要になる。

### 2. 主張 (Claim)
- **パーソナルAI/エージェントは“作る”より、権限・データ・検査点の設計で勝敗が決まる。**
- 理由: 構築のハードルは下がっても、運用（権限やルールの設計）の重要度は変わらない。むしろ、簡単に作れるからこそリスク管理が必須になる。

### 3. 具体的な論点 (Specifics)
以下の3点を中心に記事を構成します。

#### A. 権限設計（Permission Engineering）
- **範囲**: 「書き込み/送信/実行」の範囲を先に絞る。
- **リスク**: メールの自動送信、SNSへの投稿、サーバーのシャットダウンなど、破壊的な副作用を防ぐ。
- **具体例**: APIのアクセス権限（Read-only vs Read-Write）を区別し、エージェントにはまずは「参照のみ」を渡すのが安全。

#### B. データ設計（Data Context Engineering）
- **ナレッジ**: 自分の経験や知識（Obsidian, Notionなど）をどのようにエージェントに渡すか。
- **更新ルール**: 静的な知識だけでなく、日々の活動（カレンダー、タスク）をどう同期するか。
- **プライバシー**: センシティブな情報を誤って学習させないためのフィルタリング。

#### C. 検査点（Checkpoints & Security）
- **プロセス**: 重要操作の前には必ず止まる。
- **承認フロー**: たとえAIが「完璧」だと思っても、最終承認（Commit, Public）は必ず人間が行う。
- **暴走対策**: 無限ループに入った場合の強制停止手段を用意しておく。

## 執筆手順 (Workflow)

1.  **リサーチ**: パーソナルAI（ChatGPT Custom Instructions, GPTs, Claude Projects）の実例や、失敗事例（プロンプトインジェクションなど）を調べる。
2.  **型作成**: 自分なりの「個人エージェント運用ガイドライン（テンプレート）」を作成し、記事で紹介する。
3.  **記事執筆**: 「作り方」に偏りがちな技術記事へのアンチテーゼとして、運用視点（Ops）の重要性を説く。
4.  **レビュー**: 「運用の型」という言葉が、読者の「AIを使いこなしたい欲求」にマッチしているか確認する。
